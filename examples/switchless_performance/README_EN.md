# Switchless

## Overview

The switchless solution uses shared memory to reduce the number of data copies during interaction between the TEE and REE. The called data is written to the shared memory. The TEE accesses and monitors the shared memory. If any change is detected, the thread processes the change. This reduces the overhead of context switch and data copy, greatly improving performance.

## Switchless Development Process

For details about how to develop applications based on secGear APIs, see [hello_world_examples](../../README.md).

The following uses the [switchless](../switchless) sample source code as an example to describe the switchless development procedure.

### 1. Compiling the EDL API File

The following defines the ecall function test_toupper.

For details, see [switchless_edl](./switchless.edl).

```sh
    enclave {
        include "secgear_urts.h"
        from "secgear_tstdc.edl" import *;
        trusted {
            public void test_toupper([in, out, size=len]char *buf, int len) transition_using_threads;
        };
    };
```

Add the **transition_using_threads** flag to the switchless function.

### 2. Compiling REE Code

You need to perform the following steps in the REE:

- Call cc_enclave_create to create an enclave.
- Call cc_malloc_shared_memory to create the shared memory.
- Call cc_register_shared_memory to register the shared memory.
- Call the ecall function.
- Call cc_unregister_shared_memory to deregister the shared memory.
- Call cc_free_shared_memory to free the shared memory.
- Call cc_enclave_destroy to destroy the enclave.

For details, see [switchless_main.c](./host/main.c).

```sh
    // Create an enclave.
    ret = cc_enclave_create(real_p, AUTO_ENCLAVE_TYPE, 0, SECGEAR_DEBUG_FLAG, features, 1, &g_enclave);
    ...

    // Create the shared memory.
    char *buf = (char *)cc_malloc_shared_memory(&g_enclave, len);
    ...

    // Register the shared memory.
    ret = cc_register_shared_memory(&g_enclave, buf);
    ...

    // Call the ecall function. The corresponding TEE function is in enclave/enclave.c.
    test_toupper(&g_enclave, buf, strlen(TEST_STR));
    ...

    // Deregister the shared memory.
    ret = cc_unregister_shared_memory(&g_enclave, buf);
    ...

    // Free the shared memory.
    ret = cc_free_shared_memory(&g_enclave, buf);
    ...

    // Destroy the enclave.
    res = cc_enclave_destroy(context);
```

### 3. Calling the codegen Tool

For details, see [switchless_host/cmakelists.txt](./host/CMakeLists.txt).

The codegen calling has been integrated into the compilation project of the switchless sample, as described below:

```sh    
    if(CC_SGX)
        set(AUTO_FILES ${CMAKE_CURRENT_BINARY_DIR}/${PREFIX}_u.h ${CMAKE_CURRENT_BINARY_DIR}/${PREFIX}_u.c)
        add_custom_command(OUTPUT ${AUTO_FILES}
            DEPENDS ${CURRENT_ROOT_PATH}/${EDL_FILE}
            COMMAND ${CODEGEN} --${CODETYPE}
                            --untrusted ${CURRENT_ROOT_PATH}/${EDL_FILE}
                            --search-path /usr/include/secGear
                            --search-path ${SDK_PATH}/include)
    endif()
```

### 4. Compiling TEE Code

You need to perform the following step in the TEE:

- Implement the ecall function defined in the EDL file, which is equivalent to a header file.

For details, see [switchless_enclave.c](./enclave/enclave.c).

`test_t.h`: This header file is generated by the codegen tool based on the EDL file. The header file is named after the EDL file name plus "_t".

### 5. Calling the Signing Tool

See [switchless_enclave/cmakelists.txt](./enclave/CMakeLists.txt).

Use SIGN_TOOL to sign the compiled `.so` file.

## Switchless APIs

### Function APIs

- Host-side APIs

|  API  | Description |
|  ----  | ----  |
| cc_malloc_shared_memory()  | Creates the shared memory.|
| cc_free_shared_memory()  | Frees the shared memory.|
| cc_register_shared_memory()  | Registers the shared memory.|
| cc_unregister_shared_memory() | Deregisters the shared memory.|
