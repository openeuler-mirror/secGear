# Switchless

## Overview

 **Definition**: Switchless is a technology that uses shared memory to reduce the number of context switches and data copies, and optimize the interaction performance between the REE and TEE.

 **Typical application scenarios**: After a traditional application is split into a CA in the REE and a TA in the TEE for confidential computing reconstruction:

- When the CA frequently calls the TA API, the calling duration is long, which severely affects the service performance.
- When large-block data is frequently exchanged between the CA and TA, multiple memory copies are generated during the enclave call (ECALL), deteriorating the performance.
  In the preceding two typical scenarios, switchless can be used to optimize interaction performance and reduce performance loss caused by confidential computing reconstruction. The optimal effect can be the same performance as that before the reconstruction.

 **Supported hardware platforms:**

- Intel SGX
- Arm TrustZone Kunpeng 920

## Switchless Development Process

For details about how to develop applications based on secGear APIs, see [hello_world_examples](../../README.md).

The following uses the [switchless](../switchless) sample source code as an example to describe the switchless development procedure.

### 1. Compiling the EDL API File

The following defines the ecall function get_string_switchless.

For details, see [switchless_edl](./switchless.edl).

```sh
    enclave {
        include "secgear_urts.h"
        from "secgear_tstdc.edl" import *;
        from "secgear_tswitchless.edl" import *;
        trusted {
            public int get_string_switchless([out, size=32]char *buf) transition_using_threads;
        };
    };
```

Add the **transition_using_threads** flag to the switchless function.

### 2. Compiling REE Code

You need to perform the following steps in the REE:

- Call cc_enclave_create to create an enclave.
- Call cc_malloc_shared_memory to create the shared memory.
- Call the ecall function.
- Call cc_free_shared_memory to free the shared memory.
- Call cc_enclave_destroy to destroy the enclave.

For details, see [switchless_main.c](./host/main.c).

```sh
    // Create an enclave.
    res = cc_enclave_create(real_p, AUTO_ENCLAVE_TYPE, 0, SECGEAR_DEBUG_FLAG, &features, 1, context);
    ...

    // Create the shared memory.
    char *shared_buf = (char *)cc_malloc_shared_memory(context, BUF_LEN);
    ...

    // Call the ecall function. The corresponding TEE function is in enclave/enclave.c.
    res = get_string_switchless(context, &retval, shared_buf);
    ...

    // Free the shared memory.
    res = cc_free_shared_memory(context, shared_buf);
    ...

    // Destroy the enclave.
    res = cc_enclave_destroy(context);
```

There are two changes in calling the ecall function in [asynchronous_switchless_call](../switchless_performance/host/main.c):

- Initiate an asynchronous call.

```sh
    // Call the asynchronous ecall function. The corresponding TEE function is in enclave/enclave.c.
    res = get_string_switchless_async(context, &task_id, &retval, shared_buf);
    ...
```

- Query the asynchronous call result.

```sh
    // Query the asynchronous call result based on the task_id returned in step 1.
    ret = cc_sl_get_async_result(context, task_id, &retval);
    ...
```

When cc_enclave_create is called, the **ENCLAVE_FEATURE_SWITCHLESS** parameter corresponding to the switchless feature needs to be passed to use the switchless feature.

### 3. Calling the codegen Tool

For details, see [switchless_host/cmakelists.txt](./host/CMakeLists.txt).

The codegen calling has been integrated into the compilation project of the switchless sample, as described below:

```sh
 if(CC_SGX)
  set(AUTO_FILES ${CMAKE_CURRENT_BINARY_DIR}/${PREFIX}_u.h ${CMAKE_CURRENT_BINARY_DIR}/${PREFIX}_u.c)
  add_custom_command(OUTPUT ${AUTO_FILES}
   DEPENDS ${CURRENT_ROOT_PATH}/${EDL_FILE}
   COMMAND ${CODEGEN} --${CODETYPE}
                               --untrusted ${CURRENT_ROOT_PATH}/${EDL_FILE}
                               --search-path /usr/include/secGear
                               --search-path ${SDK_PATH}/include)
 endif()
```

### 4. Compiling TEE Code

You need to perform the following step in the TEE:

- Implement the ecall function defined in the EDL file, which is equivalent to a header file.

For details, see [switchless_enclave.c](./enclave/enclave.c).

`test_t.h`: This header file is generated by the codegen tool based on the EDL file. The header file is named after the EDL file name plus "_t".

### 5. Calling the Signing Tool

See [switchless_enclave/cmakelists.txt](./enclave/CMakeLists.txt).

Use SIGN_TOOL to sign the compiled `.so` file.

## Switchless APIs

### Function APIs

- Host-side APIs

|  API  | Description |
|  ----  | ----  |
| cc_malloc_shared_memory()  | Creates the shared memory that can be accessed by both the TEE and REE.<br>Parameters:<br>**enclave**: context handle in the TEE. The shared memory model varies according to the platform. To ensure cross-platform consistency of APIs, this parameter is used only on the Arm platform and is ignored on the SGX platform.<br>**size**: size of the shared memory.<br>Returns:<br>**NULL**: Failed to apply for the shared memory.<br>Other values: start address of the shared memory.<br>|
| cc_free_shared_memory()  | Frees the shared memory.<br>Parameters:<br>**enclave**: context handle in the TEE. The shared memory model varies according to the platform. To ensure cross-platform consistency of APIs, this parameter is used only on the Arm platform (the value of this parameter must be the same as the value of enclave passed when cc_malloc_shared_memory is called). It is ignored on the SGX platform.<br>**ptr**: shared memory address returned by cc_malloc_shared_memory.<br>Returns:<br>**CC_ERROR_BAD_PARAMETERS**: The input parameter is invalid.<br>**CC_ERROR_INVALID_HANDLE**: The enclave is invalid or the input enclave does not match the enclave corresponding to the ptr. (It takes effect only on the Arm platform. The SGX platform ignores the enclave and therefore does not check the enclave.)<br>**CC_ERROR_NOT_IMPLEMENTED**: The API is not implemented.<br>**CC_ERROR_SHARED_MEMORY_START_ADDR_INVALID**:<br>ptr is not the shared memory address returned by cc_malloc_shared_memory (valid only on the Arm platform).<br>**CC_ERROR_OUT_OF_MEMORY**: The memory is insufficient (valid only on the Arm platform).<br>**CC_FAIL**: common failure.<br>**CC_SUCCESS**: success.|
| cc_sl_get_async_result()  | Checks the asynchronous call result and releases the asynchronous call resources (valid only on the Arm platform).<br>Parameters:<br>**enclave**: context handle in the TEE<br>**task_id**: ID of the asynchronous call task<br>**retval**: buffer for receiving the return value<br>Returns:<br>**CC_SUCCESS**: The asynchronous call is successful.<br>**CC_ERROR_SWITCHLESS_ASYNC_TASK_UNFINISHED**: The asynchronous call is being processed.<br>**CC_ERROR_SWITCHLESS_INVALID_TASK_ID**: The task ID is invalid.<br>Other values: common failure.|
